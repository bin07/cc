4.设置文件权限
6.建共用目录（SGID）
访问权限：

基本权限

C 读取(r):允许查看内容-read
C 写入(w):允许修改内容-write
C 可执行(x):允许运行和切换-excute

#目录
    ** 若对目录有r权限，表示可列出该目录内容
    ** 若对目录有w权限，表示可在该目录下新建/删除/移动文件或子目录
    ** 若对目录有x权限，表示允许cd到该目录下
#文件
    ** 若对文件有r权限，表示可查看文件的内容
    ** 若对文件有w权限，表示可在新建/删除/移动文件内的内容
    ** 若对文件有x权限，表示可以执行这个文件

每个文件有个基本的属性
文档的归属：
C 所有者(u):拥有此文件/目录的用户-user
C 所属组(g):拥有此文件/目录的组-group
C 其他用户(o):除所有者、所属组以外的用户-other
所有用户:以上三类归属合称-all


更改访问权限：
chmod  ugo+-=rwx   文档


更改归属：
chown    属主 文件.....
         属主：属组  文件......
             ：属组  文件......
常用选项：-R 递归修改权限

#cp /etc/passwd  a.txt
#chmod u+x a.txt
#chmod o-r a.txt
#chmod u+x，o-r  a.txt
#chmod ug=rwx a.txt

权限位的八进制数表示：
r=4 w=2 x=1
rwx =7

#chmod 770 a.txt
#chmod 750 a.txt

SUID权限作用：传递所有者身份
Set UID概述
C 占用属主(User)的 x 位
C 显示为 s 或 S,取决于属主是否有 x 权限
C 仅对可执行的程序有意义
C 当其他用户执行带SUID标记的程序时,具有此程序属主的身份和相应权限

t权限应用

C 适用于目录,用来限制用户滥用写入权
C 在设置了粘滞位的文件夹下,即使用户有写入权限,
也不能删除或改名其他用户文档

[root@nsd /]# mkdir /public
[root@nsd /]# chmod 777 /public
[root@nsd /]# ls -ld /public
drwxrwxrwx. 2 root root 4096 11月 29 15:31 /public
[root@nsd /]# chmod o+t /public/
[root@nsd /]# ls -ld /public

ACL权限的应用
 可以为个别用户、组设置独立的rwx权限

?setfacl命令
C 格式:setfacl [选项] u:用户名:权限 文件...
	
? 常用命令选项
C -m:定义一条ACL策略
C -x:清除指定的ACL策略
C -b:清除所有已设置的ACL策略



[root@nsd /]# mkdir /tiantan
[root@nsd /]# ls -ld /tiantan
[root@nsd /]# chmod 750 /tiantan
[root@nsd /]# ls -ld /tiantan
[root@nsd /]# useradd tc
[root@nsd /]# su - tc
[tc@nsd ~]$ cd /tiantan
[tc@nsd ~]$ exit
[root@nsd /]# setfacl -m u:tc:rx /tiantan
[root@nsd /]# getfacl /tiantan
[root@nsd /]# su - tc
[tc@nsd ~]$ cd /tiantan
[tc@nsd ~]$ exit
